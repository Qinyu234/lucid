# Architecture Design and Questions

## New Vision Summary
- Desugar code, make state explicit
- Break classes into functions, decompose inheritance
- Map inheritance tree to virtual files, edit virtual files
- Images as virtual files
- AI confidence/complexity visualization (green=good, red=bad)
- Support function duplication, typeflow analysis, state visualization
- Test-driven workspace (auto-write tests via data flow, state)
- Local APP interface (NOT web)
- Code writing: flowchart → skeleton, estimate complexity/reliability/readability via dataflow/stateflow
- Cognitive matrix can be ignored for now

## Questions for User

### Architecture Questions:
1. **Local App Framework**: Which framework should we use for the local app?
   - Options: PyQt6, PySide6, Tkinter, Dear PyGui, Custom TUI
   - Recommendation: PyQt6 or Dear PyGui for modern UI

2. **Virtual File Format**: What format should virtual files use?
   - Options: JSON, YAML, Custom binary format, In-memory only
   - Need to decide persistence strategy

3. **Desugaring Strategy**: How aggressive should desugaring be?
   - Should we preserve original code structure?
   - Should desugared code be editable?
   - How to handle language-specific features?

4. **Complexity Metrics**: Which complexity metrics to use?
   - Cyclomatic complexity
   - Cognitive complexity
   - Semantic complexity
   - AI confidence score
   - Custom metrics?

5. **Test Generation Strategy**: How to generate tests from dataflow/state?
   - Property-based testing (Hypothesis)
   - Unit test generation
   - Integration test generation
   - Edge case detection

6. **Flowchart to Skeleton**: How to map flowchart to code skeleton?
   - Should flowchart be visual only or editable?
   - How to handle branching logic?
   - How to handle loops?

### Implementation Questions:
7. **Incremental Migration**: Should we remove web UI first or build new features in parallel?
   - Option A: Remove web UI, then build new features
   - Option B: Build new features alongside, then remove web UI

8. **Backward Compatibility**: Do we need to keep any web UI features?
   - API endpoints for external tools?
   - Command-line interface?

9. **Data Storage**: Where to store virtual files and state?
   - Local filesystem
   - In-memory database (SQLite)
   - No persistence (runtime only)

10. **AI Integration**: How will AI be integrated for code writing?
    - Local LLM (Ollama, etc.)
    - Cloud API (OpenAI, etc.)
    - Hybrid approach
