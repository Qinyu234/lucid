# CSF Expansion Engine 操作手册

## 概述

CSF (Canonical Structural Form) Expansion Engine 是一个用于分析和展开 Python 代码隐式结构的工具。它将代码的隐式结构关系显式化，提供结构化视图，并支持 Working Set 感知的投影运行时。

## 环境要求

- Python 3.11+
- 虚拟环境：.venv/

## 安装依赖

```bash
# 激活虚拟环境
.venv\Scripts\python.exe -m pip install --upgrade pip

# 安装依赖
.venv\Scripts\python.exe -m pip install -r project/requirements.txt
```

依赖包括：
- libcst>=1.1.0 - 结构化代码写回，保留格式/注释
- pydeps>=1.12.0 - 跨文件 module 依赖图
- pytest>=7.0.0 - 测试框架
- flask>=2.0.0 - Web UI 后端

## CLI 命令

### 1. expand - 展开源文件结构

```bash
.venv\Scripts\python.exe project/cli.py expand <source_path>
```

选项：
- `--show-mutations` - 显示 mutation_affects
- `--show-inherited` - 显示继承的虚拟节点

示例：
```bash
.venv\Scripts\python.exe project/cli.py expand project/examples/frontend_sample.py
.venv\Scripts\python.exe project/cli.py expand project/examples/frontend_sample.py --show-mutations
.venv\Scripts\python.exe project/cli.py expand project/examples/frontend_sample.py --show-inherited
```

### 2. heatmap - 显示模板约束热力图

```bash
.venv\Scripts\python.exe project/cli.py heatmap <source_path> --template <template_id>
```

可用模板：
- `pipeline_template` - Pipeline 模板
- `fetch_transform_cache_template` - Fetch-Transform-Cache 模板

### 3. generate - 从模板生成代码

```bash
.venv\Scripts\python.exe project/cli.py generate <source_path> --template <template_id>
```

## Python API 使用

### CSF Schema 操作

```python
from core.csf.schema import empty_csf, make_node, generate_node_id, make_chunk

# 创建空 CSF
csf = empty_csf('path/to/file.py')

# 创建节点
node = make_node('node1', 'function', 'my_func', {
    'path': 'file.py',
    'line_start': 1,
    'line_end': 10
})

# 生成节点 ID
node_id = generate_node_id()

# 创建 chunk
chunk = make_chunk('chunk1', ['node1', 'node2'], 'My Chunk', pressure_score=0.5)
```

### 代码解析

```python
from core.sync.parser import parse, parse_from_source, incremental_update

# 从文件解析
csf = parse('path/to/file.py')

# 从源代码字符串解析
source_code = open('path/to/file.py').read()
csf = parse_from_source(source_code, 'path/to/file.py')

# 增量更新（保留 expansion_state）
new_source = open('path/to/file.py').read()
csf_updated = incremental_update(csf, new_source, 'path/to/file.py')
```

### 结构展开

```python
from core.expansion import expand
from core.expansion.inheritance import expand_inheritance
from core.expansion.dependency import expand_dependencies
from core.expansion.nesting import flatten_nesting
from core.expansion.mutation import annotate_mutations

# 完整展开管道
csf = expand('path/to/file.py', options={
    "inheritance": True,
    "dependency_max_span": 2,
    "nesting_max_depth": 3,
    "mutations": True,
})

# 单独展开操作
csf = parse('path/to/file.py')
csf = expand_inheritance(csf)
csf = expand_dependencies(csf, max_span=2)
csf = flatten_nesting(csf, max_depth=3)
csf = annotate_mutations(csf)
```

### 嵌套分析

```python
from core.expansion.nesting_depth import calculate_nesting_depths
from core.expansion.nesting_guard import identify_guard_chains

# 计算嵌套深度
depths = calculate_nesting_depths(csf)

# 识别 guard 链
identify_guard_chains(csf)
```

### 代码写回

```python
from core.sync.writer import extract_function, add_early_return

# 提取函数
result = extract_function(csf, ['node1', 'node2'], 'new_function', 'path/to/file.py')
print(result['preview'])  # 操作预览
print(result['patch'])     # unified diff

# 添加 early return
result = add_early_return(csf, 'block_node_id', 'path/to/file.py')
```

### Working Set 状态计算

```python
from core.recognition.rpcm import compute_ws_state, WS_THRESHOLDS

csf = expand('path/to/file.py')
ws = compute_ws_state(csf)

print(f"R0: {ws['R0']['ws_op']}")
print(f"R1: {ws['R1']['ws_op']}")
print(f"R2: {ws['R2']['ws_op']}")
print(f"R3: {ws['R3']['ws_op']}")
print(f"Thrash risk: {ws['thrash_risk']}")

# 调整阈值
WS_THRESHOLDS["R0_nesting_depth"]["compress"] = 4
```

### 投影调度

```python
from core.projection.scheduler import schedule
from core.projection.scheduler_helpers import calculate_nesting_depth, calculate_dependency_span

csf = expand('path/to/file.py')
csf_scheduled = schedule(csf)

# 辅助函数
depth = calculate_nesting_depth(node, nodes)

```

### 模式匹配

```python
from core.pattern.matcher import match_patterns, match_pattern
from core.pattern.patterns import BUILTIN_PATTERNS

csf = expand('path/to/file.py')
matches = match_patterns(csf)

for match in matches:
    print(f"Pattern: {match['pattern_id']}")
    print(f"Node: {match['node_id']}")
    print(f"Confidence: {match['confidence']}")
    print(f"Evidence: {match['evidence']}")

# 单个 pattern 匹配
node = csf['nodes']['some_node_id']
pattern = BUILTIN_PATTERNS[0]
result = match_pattern(node, pattern, csf['nodes'])
```

from code_analysis.debugging.time_travel import TimeTravelDebugger

### 异步模式分析

分析 async/await 模式，识别潜在问题。

#### 功能

### 投影服务功能

#### WS Budget 控制
控制 Working Set 大小，超过预算自动压缩：
- MAX_R0: 最大结构节点数 (12)
- MAX_R1_CHAIN: 最大执行链长度 (5)
- MAX_R2_PROPAGATION: 最大传播深度 (4)

#### Focus Lock 机制
锁定焦点节点，冻结无关节点：
- 计算相关节点（依赖链、被依赖链、兄弟节点）
- 冻结无关节点以减少认知负载

#### Progressive Reveal
渐进式揭示 CSF 层：
- 按层级逐步揭示节点
- 每层最多显示 8 个节点

#### Projection Mode 切换
根据用户目标切换投影模式：
- `execution` - 执行模式（显示执行链、事件传播）
- `mutation` - 变异模式（仅显示变异相关节点）
- `architecture` - 架构模式（仅显示类和方法）

#### Mermaid 图表生成
支持多种图表类型：
- `structure` - 结构图
- `dataflow` - 数据流图
- `stateflow` - 状态流图
- `eventflow` - 事件流图

## 内置 Pattern

系统支持 13 个内置结构模式：

### 数据流模式

1. **pipeline** - 流水线模式
   - 线性变换链，A→B→C→D，无分支
   - 每个函数的输出是下一个的输入
   - 最少 2 个函数

2. **branch** - 分支模式
   - 条件分支执行路径
   - 有分支结构
   - 最少 2 个分支

3. **fetch_transform_cache** - 获取-转换-缓存模式
   - 外部获取 → 处理 → 本地存储
   - 有外部调用依赖
   - 有赋值（cache）
   - 有转换步骤

### 控制流模式

4. **validation_chain** - 验证链模式
   - 连续 guard clause，early return 形式
   - 有 guard_chain=True 的块
   - 最少 2 个 guard

5. **event_handler** - 事件处理器模式
   - 接收输入 → 验证 → 处理 → 响应
   - 有分支结构
   - 有变异
   - 有返回语句

### 设计模式

6. **singleton** - 单例模式
   - 类中只有一个实例
   - 有私有构造函数
   - 有静态实例方法
   - 有实例变量

7. **factory** - 工厂模式
   - 创建对象的工厂类
   - 有创建方法
   - 返回不同类型
   - 有工厂类

8. **observer** - 观察者模式
   - 事件订阅和通知机制
   - 有订阅方法
   - 有通知方法
   - 有观察者列表

9. **strategy** - 策略模式
   - 可互换的算法族
   - 有策略接口
   - 有多个实现
   - 有上下文类

10. **decorator** - 装饰器模式
    - 动态添加功能
    - 有装饰器函数
    - 有包装函数
    - 保持接口不变

11. **repository** - 仓储模式
    - 数据访问抽象层
    - 有 CRUD 方法
    - 有数据映射
    - 有抽象层

12. **service_layer** - 服务层模式
    - 业务逻辑层
    - 有业务方法
    - 无直接数据库访问
    - 有服务类

13. **command** - 命令模式
    - 命令封装
    - 有执行方法
    - 有撤销方法
    - 有命令接口
