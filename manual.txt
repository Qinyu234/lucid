# CSF Expansion Engine 操作手册

## 概述

CSF (Canonical Structural Form) Expansion Engine 是一个用于分析和展开 Python 代码隐式结构的工具。它将代码的隐式结构关系显式化，提供结构化视图。

**新愿景**：
- Desugar 代码
-使 state 显式化
- 将 class 拆解为函数，函数拆解继承
- 继承树映射到虚拟文件，编辑虚拟文件
- 图像作为虚拟文件
- AI 置信度/复杂度可视化（绿色=好，红色=差）
- 支持函数重复、typeflow 分析、state 可视化
- 通过 data flow、state 自动生成测试
- 本地 APP 界面（非网页）

## 使用方式

### GUI 应用（推荐）

启动本地 GUI 应用：

```bash
python project/app/main.py
```

GUI 应用提供：
- **代码视图** - 查看带复杂度颜色指示的代码（绿/黄/红）
- **流程图** - 可视化状态流图
- **测试** - 查看从流分析生成的测试
- **虚拟文件** - 浏览从继承树创建的虚拟文件系统

### 构建 EXE

构建独立的 exe 文件：

```bash
# 安装依赖
pip install -r project/requirements.txt

# 构建 exe
python project/build_exe.py

# exe 将创建于: project/dist/CSF_App.exe
```

### CLI 命令

#### 1. expand - 展开源文件结构

```bash
.venv\Scripts\python.exe project/cli.py expand <source_path>
```

选项：
- `--show-mutations` - 显示 mutation_affects
- `--show-inherited` - 显示继承的虚拟节点

#### 2. desugar - Desugar 代码

```bash
.venv\Scripts\python.exe project/cli.py desugar <source_path>
```

将类转换为函数，使 state 显式化。

#### 3. analyze - 分析流

```bash
.venv\Scripts\python.exe project/cli.py analyze <source_path>
```

分析 typeflow、stateflow 和 dataflow。

#### 4. complexity - 可视化复杂度

```bash
.venv\Scripts\python.exe project/cli.py complexity <source_path>
```

添加复杂度分数和颜色指示器。

#### 5. vfs - 创建虚拟文件系统

```bash
.venv\Scripts\python.exe project/cli.py vfs <source_path>
```

从继承树创建虚拟文件系统。

#### 6. test - 生成测试

```bash
.venv\Scripts\python.exe project/cli.py test <source_path>
```

从流分析自动生成测试。

## Python API 使用

### CSF Schema 操作

```python
from core.csf.schema import empty_csf, make_node, generate_node_id, make_chunk

# 创建空 CSF
csf = empty_csf('path/to/file.py')

# 创建节点
node = make_node('node1', 'function', 'my_func', {
    'path': 'file.py',
    'line_start': 1,
    'line_end': 10
})

# 生成节点 ID
node_id = generate_node_id()

# 创建 chunk
chunk = make_chunk('chunk1', ['node1', 'node2'], 'My Chunk', pressure_score=0.5)
```

### 代码解析

```python
from core.sync.parser import parse, parse_from_source, incremental_update

# 从文件解析
csf = parse('path/to/file.py')

# 从源代码字符串解析
source_code = open('path/to/file.py').read()
csf = parse_from_source(source_code, 'path/to/file.py')

# 增量更新（保留 expansion_state）
new_source = open('path/to/file.py').read()
csf_updated = incremental_update(csf, new_source, 'path/to/file.py')
```

### 结构展开

```python
from core.expansion import expand
from core.expansion.inheritance import expand_inheritance
from core.expansion.dependency import expand_dependencies
from core.expansion.nesting import flatten_nesting
from core.expansion.mutation import annotate_mutations

# 完整展开管道
csf = expand('path/to/file.py', options={
    "inheritance": True,
    "dependency_max_span": 2,
    "nesting_max_depth": 3,
    "mutations": True,
})

# 单独展开操作
csf = parse('path/to/file.py')
csf = expand_inheritance(csf)
csf = expand_dependencies(csf, max_span=2)
csf = flatten_nesting(csf, max_depth=3)
csf = annotate_mutations(csf)
```

### 嵌套分析

```python
from core.expansion.nesting_depth import calculate_nesting_depths
from core.expansion.nesting_guard import identify_guard_chains

# 计算嵌套深度
depths = calculate_nesting_depths(csf)

# 识别 guard 链
identify_guard_chains(csf)
```

### 代码写回

```python
from core.sync.writer import extract_function, add_early_return

# 提取函数
result = extract_function(csf, ['node1', 'node2'], 'new_function', 'path/to/file.py')
print(result['preview'])  # 操作预览
print(result['patch'])     # unified diff

# 添加 early return
result = add_early_return(csf, 'block_node_id', 'path/to/file.py')
```
